\subsection{Behavioral Watermarking for Agents}
\label{sec:related-behavioral}
% ============================================================
Behavioral watermarking for LLM agents embeds an owner-controlled signal into the agent's decision distribution so that ownership or origin can later be verified from the agent's behavior. AgentMark~\cite{agentmark}, ActHook~\cite{acthook}, and AGENTWM~\cite{agentwm} instantiate this idea at the visible-action layer, biasing planning choices, tool calls, or generated trajectories toward a keyed pattern that a verifier can detect by replaying or observing the action stream. \kgmark{}~\cite{kgmark} moves the carrier into structured state by watermarking dynamic knowledge graphs, which remains tied to strict knowledge graph structure.

\subsection{Long-Term Memory Systems for Agents}
\label{sec:related-memory}
% ============================================================
Long-term memory systems give LLM agents a persistent, evolving state that outlives a single context window. \amem{}, \graphiti{}, MemOS, Memory-R1, and MemMachine~\cite{amem,graphiti,memos,memmachine,yan2025memory} represent the current generation, spanning note-network, knowledge-graph, and structured-cube designs, and a subset of them, including \tiermem{} and MemOS~\cite{tiermem,memos}, further attach source anchoring, versioning, and lifecycle metadata to individual memory items as a form of stored provenance. Parallel work studies how these systems can be attacked: A-MemGuard and MemoryGraft~\cite{amemguard,memorygraft} demonstrate poisoning and stealth-edit attacks against long-term memory.
