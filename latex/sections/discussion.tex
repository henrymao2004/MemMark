\paragraph{Why state-evolution, not action.}
\todo{Summarize the trajectory-vs-state argument from \S\ref{sec:intro}
plus the empirical R3 vs Action-layer comparison from

\paragraph{What the distribution-preservation guarantee buys.}
\todo{Lemma \ref{lem:dist-preserve} is a per-decision marginal
guarantee. Empirically (\S\ref{sec:rq1}), F1 deltas across LLMs
fall within statistical noise with inconsistent sign — consistent
with the guarantee. The \texttt{signed-metadata-only} baseline
matches \texttt{no-watermark} F1, isolating the cost of the
keyed-pick selection itself from the audit-trace machinery around
it.}

\paragraph{Limitations.}
\todo{(i) Backend coverage limited to \amem{} / \graphiti{};
extending to MemOS / MemMachine is engineering. (ii) Adversary
that can both write the snapshot \emph{and} read the secret key
trivially forges; we assume the key is not exfiltrated.
(iii) cat-5 adversarial questions remain difficult; F1 lifted to
$\approx 0.4$ after the cat-5 loader fix but still below paper-level
for some backends. (iv) Update / link / merge / delete / stale-memory
/ temporal-consistency integrity require LongMemEval-style
ground-truth labels and are not reported in this draft.}

\paragraph{Future work.}
\todo{(i) Larger $K$ — current self-reported enumeration is $K=4$;
$K{=}8$ doubles per-decision capacity but cost / quality trade-off
needs study. (ii) Online attestation for streaming agents.
(iii) Cross-session anchor chaining for long-running deployments.}
