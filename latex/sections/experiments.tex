\subsection{RQ3 — Snapshot-Only / Partial-Log Verification (Headline)}
\label{sec:rq3}
% ============================================================
\paragraph{Why this is the headline.} \todo{In the most forensically
valuable setting — memory leaked / distilled / migrated — the
action trajectory is gone. RQ3 tests whether \memmark{} can still
attribute. This is the central claim of the paper.}

\paragraph{Three regimes.} \todo{R1 = full external log (audit
upper bound). R2 = partial log with keep ratio $r \in \{0.1, 0.3,
0.5, 0.7, 0.9\}$. R3 = snapshot only, using the in-record reveal
sidecar and the per-leaf inclusion proof of
\S\ref{sec:method-audit}.}

\input{tables/verification}

\paragraph{Headline result.} \todo{R3 \texttt{bit\_recovery} reaches
$1.00$ with \texttt{anchor\_signature\_valid}; the
\texttt{signed-metadata-only} ablation drops to $0$ (writer cannot
be identified from signed metadata alone — exactly the failure
mode this paper targets); the wrong-key control drops to $\approx
0.15$, below the random-pick floor $1/K \approx 0.33$. R2 degrades
smoothly with $r$ — see Figure~\ref{fig:r2-curve}. \memmark{} is
the only point in the regime matrix that succeeds at R3.}

\input{figures/r2_curve}

\paragraph{R3 = 1.00 sanity checks.} \todo{Three orthogonal
controls rule out a trivial verifier returning constant 1.0:
(a) wrong-key gives $0.15 \ll 1/K \approx 0.33$, proving the
verifier depends on the secret key; (b) \texttt{signed-metadata-only}
gives $0$ under the same R3 protocol, proving the bit-decoding
step is actually executed; (c) the R2 sweep is strictly monotone
in keep ratio, proving the verifier counts kept-leaf bits rather
than short-circuiting.}

\input{tables/robustness}

\paragraph{Per-leaf inclusion proof keeps bit recovery smooth.}
\todo{Method B (\S\ref{sec:method-audit}) stores each leaf's
Merkle path at seal time, so structural removal (pruning, dedup,
poisoning, compaction) does not invalidate surviving leaves.
\texttt{bit\_recovery\_post} stays at $1.00$ under pruning at all
three strengths and the removal itself is reported via
\texttt{missing\_leaves\_rate}. Content-tamper attacks
(manual\_edits, paraphrase\_rewrite, edge\_relabel,
subgraph\_reanchor, supersession) show linear degradation in
\texttt{bit\_recovery\_post} matched by linear
\texttt{commitment\_fail\_rate}. See
