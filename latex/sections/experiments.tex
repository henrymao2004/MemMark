\subsection{RQ3 — Snapshot-Only / Partial-Log Verification (Headline)}
\label{sec:rq3}
% ============================================================
\paragraph{Why this is the headline.} \todo{In the most forensically
valuable setting — memory leaked / distilled / migrated — the
action trajectory is gone. RQ3 tests whether \memmark{} can still
attribute. This is the central claim of the paper; everything that
follows (robustness, capacity, utility trade-off) supports or
quantifies the price of this guarantee.}

\paragraph{Three regimes.} \todo{R1 = full external log (audit
upper bound). R2 = partial log with keep ratio $r \in \{0.1, 0.3,
0.5, 0.7, 0.9\}$. R3 = snapshot only, using the in-record reveal
sidecar and the per-leaf inclusion proof of
\S\ref{sec:method-audit}.}

\input{tables/verification}

\paragraph{Headline result.} \todo{R3 \texttt{bit\_recovery} reaches
$1.00$ with \texttt{anchor\_signature\_valid}; the
\texttt{signed-metadata-only} ablation drops to $0$ (writer cannot
be identified from signed metadata alone — exactly the failure
mode this paper targets); the wrong-key control drops to $\approx
0.15$, below the random-pick floor $1/K \approx 0.33$. R2 degrades
smoothly with $r$ — see Figure~\ref{fig:r2-curve}. \memmark{} is
the only point in the regime matrix that succeeds at R3.}

\subsection{RQ4 — Robustness Against Memory-Specific Attacks}
\label{sec:rq4}
% ============================================================
\todo{Nine attacks $\times$ three strengths $(0.1, 0.3, 0.5)$
applied to the watermarked snapshot, then re-run R3 verification.
The attacks split cleanly along the audit layer they trigger:
content-tamper attacks fire \texttt{commitment\_fail\_rate};
structural attacks fire \texttt{missing\_leaves\_rate}.}

\input{tables/robustness}

\paragraph{Per-leaf inclusion proof keeps bit recovery smooth.}
\todo{Method B (\S\ref{sec:method-audit}) stores each leaf's
Merkle path at seal time, so structural removal (pruning, dedup,
poisoning, compaction) does not invalidate surviving leaves. As a
result \texttt{bit\_recovery\_post} stays at $1.00$ under pruning
at all three strengths (every kept leaf still verifies) and the
removal itself is reported via \texttt{missing\_leaves\_rate}.
Content-tamper attacks (manual\_edits, paraphrase\_rewrite,
edge\_relabel, subgraph\_reanchor, supersession) show smooth
linear degradation in \texttt{bit\_recovery\_post} matched by
linear \texttt{commitment\_fail\_rate} — exactly the desired
behavior. See Figure~\ref{fig:attack-signal-split}.}

\subsection{RQ1 — Controlled Utility Trade-off}
\label{sec:rq1}
% ============================================================
\todo{We are not claiming utility preservation; we are quantifying
the cost of the RQ3 guarantee. Watermark embedding biases keyword
/ tag / context choices away from the LLM's argmax, which slightly
degrades downstream retrieval recall (RQ5
\texttt{evidence\_recall\_mean}), which in turn slightly degrades
QA F1. We measure both endpoints and report the trade-off
explicitly.}

\input{tables/tradeoff_comparison}

\paragraph{Per-benchmark numbers.} \todo{On \locomo{} the relative
F1 drop is $\approx 6{-}7\%$; on \longmemeval{} and \mab{}
\todo{numbers TBD with multi-seed runs}. Per-category breakdown
(Table~\ref{tab:main-results}) shows the cost is uniform across
the five QA categories with no pathological category-level
collapse.}

\input{tables/main_results}
\input{figures/per_category}

\paragraph{Why this trade-off shape is the right one.} \todo{The
\texttt{signed-metadata-only} baseline matches \texttt{no-watermark}
F1 to within statistical noise, which proves the audit-trace
machinery itself is utility-neutral. The $\sim$7\% gap is therefore
attributable to the keyed bit-embedding alone, and it buys the
$1.00$ snapshot-only attribution that no other method in
Table~\ref{tab:tradeoff} provides. Section~\ref{sec:discuss}
discusses why retrieval-aware sampling is a natural follow-up to
narrow this gap.}
